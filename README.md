# Описание метода моделирования, на основании которого конструируется дискретно-событийный монитор
В данной работе применяется императивный метод дискретно-событийного моделирования, в котором центральную роль играет календарь событий и система из нескольких таймеров с относительными шкалами времени. Управляющие операторы модели изменяют состояние календарей, планируют будущие события и обеспечивают продвижение модельного времени.  
В системе существует один системный таймер, задающий модельное время. Кроме него определены два таймера, чьи показания представляют собственные локальные шкалы времени. Эти шкалы задаются через смещения (offset) относительно системного времени. Таким образом, каждый таймер имеет локальное отображаемое время, но все события переводятся в абсолютное время через величину смещения.  
Планирование события осуществляется следующим образом:
- пользователь задаёт время события в локальной шкале таймера;
- модель переводит это время в абсолютное;
- абсолютное время всегда ≥ 0 и выражает реальный момент наступления события в общей временной оси;
- событие включается в соответствующий календарь.  

Для активированных типов событий дальнейшие появления генерируются автоматически по экспоненциальному закону, и каждое последующее событие немедленно планируется в календарь.  
Модельное время отслеживается системным таймером и изменяется в ходе обработки событий. При запуске имитации на заданный интервал времени выполняется:
- выбор ближайшего по абсолютному времени события из всех таймеров (просмотр глобального календаря);
- проверка попадания события в окно интервала;
- продвижение системного времени на заданный интервал;
- выполнение действия события;
- удаление события из локального календаря;
- автоматическое планирование следующего события данного типа.  

Хотя каждый таймер имеет собственный календарь, глобальный диспетчер событий объединяет их в общий хронологический порядок. Это позволяет выполнять обработку событий в правильной временной последовательности, учитывать переходы через границу суток, корректно согласовывать локальные времена с абсолютной шкалой.

# Описание классов, которые разработаны для реализации дискретно-событийного монитора
Класс EventNode. Представляет одно событие календаря (узел двусвязного циклического списка).  
Класс EventType. Информация о типе событий (имя события, параметры генерации следующих событий в самопорождающемся процессе, путь к изображению).  
Класс TimerNode. Представляет таймер и его локальный календарь событий — двусвязный циклический список EventNode. Для каждого таймера хранится оффсет относительно системного времени, имя и map картинок. Есть методы, реализующие вставку события в календарь в порядке возрастания, удаление события.  
Класс Monitor. Содержит системное время, три таймера (SystemTimer, T1, T2), реестр типов событий и алгоритмы глобальной обработки событий (поиск следующего, обработка интервала, шаг по системному времени, вызывает порождение следующего события).  
Класс TimerForm – форма отдельного таймера. Отображает список локальных событий, позволяет выбрать тип события и задать локальное время. Поддерживает удаление локальных событий.  
DigitizeForm – форма для установки SystemTime и offset-ов (оцифровка).  
GlobalForm показывает глобальный календарь.
